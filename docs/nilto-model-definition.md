# NILTO モデル定義：トップページ（index）コンテンツ移行

`src/pages/index.astro` のコンテンツを NILTO に移行するためのモデル定義です。

---

## 1. 現在の index.astro のデータ

| 変数 | 型 | 現在の値例 |
|------|-----|------------|
| hamu_name | string | ぽこちゃん🐹 |
| description | string | さいきん　いえに　やってきた　ハムスターだよ㊗️ |
| skills | string[] | ひまわりのタネ, 観覧車, もごもご, ねんね |
| 画像 | ローカルアセット | poko-01.jpeg |
| ページタイトル | string | わが家の新人ハムスター |

---

## 2. モデル定義

### モデル（コンテンツタイプ）

| 項目 | 値 | 説明 |
|------|-----|------|
| **モデル LUID** | `top_page` | 英小文字・数字・アンダースコアのみ。スペース内でユニーク |
| **表示名** | トップページ | 管理画面用 |

※ LUID は NILTO 管理画面「モデル詳細 > 基本情報ダイアログ」で設定・確認できます。

---

### フィールド一覧

| 順序 | フィールドLUID | 表示名 | フィールド種類 | 必須 | 説明・入力値 |
|------|----------------|--------|----------------|------|--------------|
| 1 | `hamu_name` | ハムスターの名前 | **1行テキスト** | ○ | 名前＋絵文字用。例: `ぽこちゃん🐹` |
| 2 | `description` | 説明文 | **複数行テキスト** | ○ | 自己紹介文。例: `さいきん　いえに　やってきた　ハムスターだよ㊗️` |
| 3 | `main_image` | メイン画像 | **メディア** | △ | ハムスターの写真。NILTO にアップロードした画像を選択。alt 例: `ぽこちゃん` |
| 4 | `skills` | とくいなこと（繰り返し） | **繰り返し** | - | 下の「繰り返し内フィールド」を参照 |

#### 繰り返しフィールド `skills` の内包フィールド

| フィールドLUID | 表示名 | フィールド種類 | 説明 |
|----------------|--------|----------------|------|
| `skill_name` | とくいなこと（1件） | **1行テキスト** | 例: ひまわりのタネ, 観覧車, もごもご, ねんね |

---

### システムプロパティ（NILTO 標準）

ページタイトルは **タイトルフィールド（_title）** に設定します。コンテンツ作成時に以下を指定します。

| システムキー | 用途 | 入力値例 |
|-------------|------|----------|
| `_title` | コンテンツタイトル（ページタイトル） | `わが家の新人ハムスター` |

※ 一覧表示や OGP などにも使われるため、ここにページタイトルを入れるのがおすすめです。

---

## 3. フィールド種類と NILTO 上の設定の対応

| フィールド種類（NILTO） | 対応する index.astro のデータ | API レスポンス例 |
|-------------------------|-------------------------------|------------------|
| 1行テキスト | hamu_name | `"hamu_name": "ぽこちゃん🐹"` |
| 複数行テキスト | description | `"description": "さいきん　いえに　やってきた　ハムスターだよ㊗️"` |
| メディア | 画像（要アップロード） | `"main_image": { "url": "https://...", "alt": "ぽこちゃん" }` |
| 繰り返し（内包: 1行テキスト） | skills 配列 | `"skills": [ { "skill_name": "ひまわりのタネ" }, ... ]` |

---

## 4. 初回コンテンツの入力値（登録用）

NILTO で「トップページ」モデルのコンテンツを 1 件作成するときの入力例です。

### 管理画面で入力する場合

- **タイトル（_title）**: `わが家の新人ハムスター`
- **hamu_name**: `ぽこちゃん🐹`
- **description**: `さいきん　いえに　やってきた　ハムスターだよ㊗️`
- **main_image**: メディアライブラリに `poko-01.jpeg` をアップロードし、それを選択。alt: `ぽこちゃん`
- **skills**（繰り返し）:
  - 1件目: `ひまわりのタネ`
  - 2件目: `観覧車`
  - 3件目: `もごもご`
  - 4件目: `ねんね`

### 書き込み API（POST）で登録する場合の JSON 例

```json
{
  "model": "top_page",
  "_title": "わが家の新人ハムスター",
  "fields": {
    "hamu_name": "ぽこちゃん🐹",
    "description": "さいきん　いえに　やってきた　ハムスターだよ㊗️",
    "main_image": {
      "url": "https://cms-assets.nilto.com/.../poko-01.jpeg",
      "alt": "ぽこちゃん"
    },
    "skills": [
      { "skill_name": "ひまわりのタネ" },
      { "skill_name": "観覧車" },
      { "skill_name": "もごもご" },
      { "skill_name": "ねんね" }
    ]
  }
}
```

※ 画像は先に Media 書き込み API でアップロードし、返却された URL を `main_image.url` に指定します。

---

## 5. 取得 API の例

トップページ用コンテンツが 1 件だけの想定の場合（ID 指定で取得）:

```
GET https://cms-api.nilto.com/v1/contents/{content_id}?model=top_page
```

複数件ある場合は一覧取得して先頭を利用:

```
GET https://cms-api.nilto.com/v1/contents?model=top_page&limit=1
```

レスポンス例:

```json
{
  "_id": 1234567890,
  "_model": "top_page",
  "_title": "わが家の新人ハムスター",
  "_created_at": "2025-03-01T00:00:00Z",
  "_updated_at": "2025-03-01T00:00:00Z",
  "_published_at": "2025-03-01T00:00:00Z",
  "_status": "published",
  "hamu_name": "ぽこちゃん🐹",
  "description": "さいきん　いえに　やってきた　ハムスターだよ㊗️",
  "main_image": {
    "url": "https://cms-assets.nilto.com/.../poko-01.jpeg",
    "alt": "ぽこちゃん"
  },
  "skills": [
    { "skill_name": "ひまわりのタネ" },
    { "skill_name": "観覧車" },
    { "skill_name": "もごもご" },
    { "skill_name": "ねんね" }
  ]
}
```

## 6. LUID ルールのまとめ

- **使用可能文字**: 英小文字 (a-z)、数字 (0-9)、アンダースコア (_) のみ
- **モデル LUID**: スペース内でユニーク（例: `top_page`）
- **フィールド LUID**: 同じモデル（または同じフィールドセット）内でユニーク
- **繰り返し内フィールド**: その繰り返しフィールド内でユニーク（例: `skill_name`）

---

## 7. 次のステップ

1. NILTO 管理画面でモデル `top_page` を作成し、上記フィールドを追加
2. メディアで `poko-01.jpeg` をアップロード
3. コンテンツを 1 件作成し、上記の入力値を登録
4. Astro 側で NILTO の GET API を呼び、取得したデータで `index.astro` をレンダリングするよう変更

必要なら、Astro 側の実装例（fetch とマッピング）も別ファイルでまとめられます。

## 8. モデル `album`（日別アルバム・カレンダー用）

撮影日ごとに 1 コンテンツを作成し、その日の画像を繰り返しフィールドに並べます。サイト側では一覧 API で取得し、トップのカレンダーと `/album/YYYY-MM-DD` のアルバムページに利用します。

### モデル（コンテンツタイプ）

| 項目 | 値 | 説明 |
|------|-----|------|
| **モデル LUID** | `album` | 英小文字・数字・アンダースコアのみ |
| **表示名** | 撮影日アルバム（任意） | 管理画面用 |

### フィールド一覧

| 順序 | フィールドLUID | 表示名 | フィールド種類 | 必須 | 説明 |
|------|----------------|--------|----------------|------|------|
| 1 | `shoot_date` | 撮影日 | **日付**（なければ **1行テキスト** で `YYYY-MM-DD`） | ○ | カレンダーと URL 用 |
| 2 | `photos` | 写真（繰り返し） | **繰り返し** | △ | その日の画像 |

#### 繰り返し `photos` の内包フィールド

| フィールドLUID | 表示名 | フィールド種類 | 説明 |
|----------------|--------|----------------|------|
| `image` | 画像 | **メディア** | 表示用 URL・alt |
| `alt` | 代替テキスト | **1行テキスト**（任意） | `image.alt` の補足 |

### 取得 API

一覧（サイトはビルド時に全件または十分な `limit` で取得し、日付でグループ化）:

```
GET https://cms-api.nilto.com/v1/contents?model=album&limit=100&offset=0
```

ヘッダ: `X-NILTO-API-KEY: <APIキー>`

**一覧レスポンスの形**は API バージョンにより配列直返却の場合と、`contents` / `data` などに配列が入る場合があります。Astro 側の `src/lib/nilto.ts` で複数形に対応しています。

### レスポンス例（1件・イメージ）

```json
{
  "_id": 9876543210,
  "_model": "album",
  "_title": "2025-03-29 のぽこちゃん",
  "_status": "published",
  "shoot_date": "2025-03-29",
  "photos": [
    {
      "image": { "url": "https://cms-assets.nilto.com/.../1.jpg", "alt": "ぽこちゃん" },
      "alt": "ぽこちゃん"
    }
  ]
}
```

※ `shoot_date` が `2025-03-29T00:00:00.000Z` 形式で返る場合も、サイト側で `YYYY-MM-DD` に正規化します。

※ **一覧 API** ではカスタムフィールドが省略される場合があります。そのときは先頭が `YYYY-MM-DD` になるよう **`_title`** を付けておくと、サイト側が日付として認識します（`shoot_date` があればそちらを優先）。

### 環境変数（Astro）

| 変数 | 必須 | 説明 |
|------|------|------|
| `NILTO_API_KEY` | 推奨 | 読み取り API キー |
| `NILTO_API_BASE` | 推奨 | 例: `https://cms-api.nilto.com/v1`（末尾スラッシュなし）。**未設定でも** `NILTO_CONTENT_URL` が `.../v1/contents/...` 形式なら、そこからベース URL を推測して `album` 一覧を取得します |
| `NILTO_CONTENT_URL` | 任意 | **トップページ単体**の GET URL。設定時は `top_page` 取得に優先。パスに `contents` が含まれる場合、`album` 一覧用の API ベース推測にも使います |
| `NILTO_ALBUM_LIST_URL` | 任意 | **案B**: `album` 一覧の完全 URL。未設定時はコード側が `limit=100` と `offset` でページング取得（公式は **limit 最大 100**） |
| `NILTO_PHOTO_DAY_LIST_URL` | 任意 | 旧名の互換。`NILTO_ALBUM_LIST_URL` と同じ（どちらか一方を設定） |

API キーだけで **`NILTO_CONTENT_URL` も `NILTO_API_BASE` も無い**、または **URL が `.../contents/...` 形式で推測できない**場合は `album` 一覧は取得できません。`NILTO_API_BASE` の明示設定が確実です。
